PenguinCAM Roadmap

Development priorities and future enhancements for PenguinCAM.

✅
Current Status: PenguinCAM is deployed and production-ready at penguincam.popcornpenguins.com

Current Features

PenguinCAM is ready for real-world FRC use with these features:

✅

Onshape Integration

One-click export from Onshape with OAuth authentication

✅

Automatic Detection

Top face detection, hole identification, and smart feature recognition

✅

Part Orientation

Rotate in 90° increments with fixed bottom-left origin

✅

3D Visualization

Interactive preview with toolpath scrubber and tool animation

✅

Multi-Layer Support

2.5D machining with grooves, pockets, and countersinks

✅

Tube Machining

Square ends and mirror-image patterns for aluminum tube

✅

Tool Compensation

Automatic offset for perimeters, pockets, and holes

✅

Smart Tabs

Automatic tab placement and removal for secure parts

✅

Google Drive

Direct upload to team shared drives

✅

Multi-Team

Team-specific configurations via YAML files

Future Enhancements

Planned features in no particular order. Priority and effort are approximate.

1

Multiple Parts Per Job

Priority: Medium | Effort: Medium-High

Layout multiple instances of the same part on a single piece of stock, or combine different parts in one cutting job. Would optimize material usage and reduce setup time. See "Interactive Part Layout Interface" for visual positioning controls.

2

Per-Team Branding

Priority: Low | Effort: Medium

Customize the UI with team colors, logos, and names from the config file. Would make PenguinCAM feel more like "your" tool.

3

Additional Cloud Storage

Priority: Low | Effort: Medium

Support for Dropbox, OneDrive, or other cloud storage services beyond Google Drive. Would benefit teams not using Google Workspace.

4

G41/G42 Tool Compensation

Priority: Low | Effort: Medium

Use controller-based tool compensation instead of pre-compensated paths. Would allow more precise finished products, especially for bearing holes where 1.125" often ends up too small.

5

Enhanced 2.5D Features

Priority: Medium | Effort: High

Status: Testing! Multi-layer DXF export for parts with partial-depth pockets, grooves, and countersinks. Uses face analysis in Onshape to automatically detect and export multiple depth layers.

6

Asymmetric Tube Patterns

Priority: Medium | Effort: High

Support tubing where opposite faces aren't the same. Many teams do tube designs with larger holes on one side to support access to bolt heads.

7

Interactive Part Layout Interface

Priority: Medium | Effort: High

Slicer-style interface for positioning parts on the machine bed. Show the full bed size in the preview, allow drag-and-drop positioning, arbitrary rotation and mirroring, and define exclusion zones for hold-downs. Would extend the multiple parts feature with intuitive visual controls familiar to students from 3D printing.

8

Intelligent Pocket Machining Strategy

Priority: Medium | Effort: Medium-High

Use area-based logic to choose between spiral milling (small pockets) and perimeter contouring with tabs (large pockets). Currently, all pockets are milled out regardless of size, which is inefficient for large features. Threshold would be configurable as a multiple of tool diameter squared.

9

Show More Gcode Text During Toolpath Preview

Priority: Low | Effort: Low

Many gcode preview tools allow you to step through the gcode text line by line while showing several lines of context. It would be good to replicate that. Useful for debugging, maybe less important for students.

Ideas for Consideration

Not committed to the roadmap yet, but worth exploring:

  • Batch processing multiple DXFs in one job
  • G-code optimization for shorter tool paths
  • Export simulation as video or animated GIF
  • Integration with other CAD platforms (Fusion 360, Inventor)
  • Library of CNC machines with pre-configured settings
  • Custom tab placement (manual positioning)
  • Tool library management
  • Estimated cycle time improvements
  • Adaptive toolpath strategies

If you're interested in contributing:

  1. Open an issue on GitHub to discuss your idea
  2. Fork the repository and make your changes
  3. Submit a pull request with tests and documentation
